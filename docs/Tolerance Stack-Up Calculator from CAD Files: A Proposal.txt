

# Tolerance Stack-Up Calculator from CAD Files: A Proposal

## 1. Problem Statement

Tolerance stack-up analysis is one of the most error-prone and time-consuming tasks in mechanical engineering. Engineers typically:
- Manually identify dimension chains from 2D drawings or 3D models
- Build spreadsheets for worst-case (WC) and statistical (RSS) analysis
- Iterate when designs change, often losing traceability
- Miss critical stack-ups until assembly failures occur

**The goal:** A tool that ingests CAD files (STEP, IGES, or native formats), identifies geometric relationships, and semi-automatically constructs tolerance stack-up chains with quantified results.

### 3.2 Dimension Chain Builder (Graph Model)

This is the **core algorithmic challenge**. The assembly is modelled as a directed graph:

- **Nodes** = part features (faces, axes, datums)
- **Edges** = internal part dimensions (within a part) or external contacts (between parts)

Finding a stack-up loop = finding a **path** from one side of a critical gap/clearance to the other, through the chain of contributing dimensions.

```python name=stack_up_graph.py
"""
Builds a graph representation of the assembly dimension chain
and identifies tolerance stack-up loops.
"""
import networkx as nx
import numpy as np
from dataclasses import dataclass, field
from typing import Optional
from enum import Enum


class ContributorType(Enum):
    INTERNAL = "internal"   # dimension within a single part
    CONTACT = "contact"     # interface between two parts
    GAP = "gap"             # the critical clearance/gap being analyzed


@dataclass
class DimensionContributor:
    """One contributor in a tolerance stack-up chain."""
    name: str
    contributor_type: ContributorType
    nominal: float           # mm, signed (+/- per direction convention)
    tol_plus: float          # mm, always positive
    tol_minus: float         # mm, always positive (magnitude)
    distribution: str = "normal"
    sensitivity: float = 1.0  # partial derivative / direction cosine
    part_name: str = ""
    description: str = ""


@dataclass
class StackUpResult:
    """Result of a tolerance stack-up calculation."""
    gap_name: str
    nominal_gap: float
    worst_case_min: float
    worst_case_max: float
    rss_min: float
    rss_max: float
    monte_carlo_mean: Optional[float] = None
    monte_carlo_std: Optional[float] = None
    monte_carlo_yield: Optional[float] = None  # % within spec
    contributors: list[DimensionContributor] = field(default_factory=list)
    sensitivity_ranking: list[tuple[str, float]] = field(default_factory=list)


class ToleranceStackUpGraph:
    """
    Models an assembly as a graph where nodes are geometric features
    and edges are dimensional contributors (internal or contact).

    The user selects two features defining a critical gap;
    the graph finds the shortest loop and computes the stack-up.
    """

    def __init__(self):
        self.graph = nx.DiGraph()

    # ── Graph Construction ──────────────────────────────────────

    def add_internal_dimension(
        self,
        part_name: str,
        feature_a_id: str,
        feature_b_id: str,
        nominal: float,
        tol_plus: float,
        tol_minus: float,
        direction_vector: np.ndarray,
        description: str = "",
    ):
        """
        Add a dimension between two features on the SAME part.
        Nominal is signed: positive in the stack-up direction.
        """
        self.graph.add_edge(
            feature_a_id, feature_b_id,
            contributor=DimensionContributor(
                name=f"{part_name}: {feature_a_id} → {feature_b_id}",
                contributor_type=ContributorType.INTERNAL,
                nominal=nominal,
                tol_plus=tol_plus,
                tol_minus=tol_minus,
                part_name=part_name,
                description=description,
            ),
            direction=direction_vector,
        )

    def add_assembly_contact(
        self,
        part_a_feature: str,
        part_b_feature: str,
        gap_nominal: float = 0.0,
        tol_plus: float = 0.0,
        tol_minus: float = 0.0,
        contact_type: str = "rigid",
    ):
        """
        Add an edge between features on DIFFERENT parts
        that are in contact at assembly.
        """
        self.graph.add_edge(
            part_a_feature, part_b_feature,
            contributor=DimensionContributor(
                name=f"Contact: {part_a_feature} ↔ {part_b_feature}",
                contributor_type=ContributorType.CONTACT,
                nominal=gap_nominal,
                tol_plus=tol_plus,
                tol_minus=tol_minus,
                description=contact_type,
            ),
        )

    # ── Loop Identification ─────────────────────────────────────

    def find_stack_up_chain(
        self, start_feature: str, end_feature: str
    ) -> list[DimensionContributor]:
        """
        Find the dimension chain (shortest path) between two features
        that define the critical gap or clearance.

        Returns ordered list of contributors from start to end.
        """
        try:
            path = nx.shortest_path(
                self.graph, source=start_feature, target=end_feature
            )
        except nx.NetworkXNoPath:
            raise ValueError(
                f"No connected dimension chain found between "
                f"'{start_feature}' and '{end_feature}'. "
                f"Check that all intermediate contacts are defined."
            )

        contributors = []
        for i in range(len(path) - 1):
            edge_data = self.graph.edges[path[i], path[i + 1]]
            contributors.append(edge_data["contributor"])

        return contributors

    # ── Stack-Up Solvers ────────────────────────────────────────

    def compute_stack_up(
        self,
        start_feature: str,
        end_feature: str,
        gap_spec_min: float = None,
        gap_spec_max: float = None,
        monte_carlo_iterations: int = 100_000,
    ) -> StackUpResult:
        """
        Full stack-up analysis: worst-case, RSS, and Monte Carlo.
        """
        contributors = self.find_stack_up_chain(start_feature, end_feature)

        # ── Nominal gap ──
        nominal_gap = sum(c.nominal * c.sensitivity for c in contributors)

        # ── Worst-Case ──
        wc_tol_upper = sum(
            max(c.tol_plus, c.tol_minus) * abs(c.sensitivity)
            for c in contributors
        )
        wc_min = nominal_gap - wc_tol_upper
        wc_max = nominal_gap + wc_tol_upper

        # ── RSS (Root Sum of Squares) ──
        # Assumes symmetric, normal distributions centered on nominal
        rss_tol = np.sqrt(sum(
            (((c.tol_plus + c.tol_minus) / 2) * c.sensitivity) ** 2
            for c in contributors
        ))
        rss_min = nominal_gap - 3 * rss_tol  # 3σ limits
        rss_max = nominal_gap + 3 * rss_tol

        # ── Monte Carlo ──
        mc_result = self._monte_carlo(
            contributors, monte_carlo_iterations,
            gap_spec_min, gap_spec_max,
        )

        # ── Sensitivity Ranking ──
        sensitivity = self._sensitivity_analysis(contributors)

        return StackUpResult(
            gap_name=f"{start_feature} → {end_feature}",
            nominal_gap=nominal_gap,
            worst_case_min=wc_min,
            worst_case_max=wc_max,
            rss_min=rss_min,
            rss_max=rss_max,
            monte_carlo_mean=mc_result["mean"],
            monte_carlo_std=mc_result["std"],
            monte_carlo_yield=mc_result["yield_pct"],
            contributors=contributors,
            sensitivity_ranking=sensitivity,
        )

    def _monte_carlo(
        self,
        contributors: list[DimensionContributor],
        n_iterations: int,
        spec_min: float = None,
        spec_max: float = None,
    ) -> dict:
        """
        Monte Carlo simulation of the stack-up.
        Each contributor is sampled from its specified distribution.
        """
        rng = np.random.default_rng(seed=42)
        samples = np.zeros(n_iterations)

        for c in contributors:
            midpoint = c.nominal + (c.tol_plus - c.tol_minus) / 2
            half_range = (c.tol_plus + c.tol_minus) / 2

            if c.distribution == "normal":
                # 3σ = half_range → σ = half_range / 3
                sigma = half_range / 3 if half_range > 0 else 1e-9
                dim_samples = rng.normal(loc=midpoint, scale=sigma,
                                         size=n_iterations)
            elif c.distribution == "uniform":
                dim_samples = rng.uniform(
                    low=midpoint - half_range,
                    high=midpoint + half_range,
                    size=n_iterations,
                )
            else:  # default to normal
                sigma = half_range / 3 if half_range > 0 else 1e-9
                dim_samples = rng.normal(loc=midpoint, scale=sigma,
                                         size=n_iterations)

            samples += dim_samples * c.sensitivity

        # Yield calculation
        yield_pct = 100.0
        if spec_min is not None and spec_max is not None:
            in_spec = np.sum((samples >= spec_min) & (samples <= spec_max))
            yield_pct = (in_spec / n_iterations) * 100.0

        return {
            "mean": float(np.mean(samples)),
            "std": float(np.std(samples)),
            "yield_pct": yield_pct,
            "histogram": samples,  # for plotting
        }

    @staticmethod
    def _sensitivity_analysis(
        contributors: list[DimensionContributor],
    ) -> list[tuple[str, float]]:
        """
        Rank contributors by their variance contribution (%).
        Tells the engineer WHERE to tighten tolerances for most effect.
        """
        variances = []
        for c in contributors:
            half_range = (c.tol_plus + c.tol_minus) / 2
            sigma = half_range / 3
            var = (sigma * c.sensitivity) ** 2
            variances.append((c.name, var))

        total_var = sum(v for _, v in variances)
        if total_var == 0:
            return [(name, 0.0) for name, _ in variances]

        ranked = [
            (name, (var / total_var) * 100)
            for name, var in variances
        ]
        ranked.sort(key=lambda x: x[1], reverse=True)
        return ranked
```

### 3.3 Tolerance Assignment Module

When PMI data is absent (common in practice), tolerances must come from somewhere:

```python name=tolerance_defaults.py
"""
Process-based default tolerances.
Used when CAD PMI annotations are missing.

Sources:
  - Machinery's Handbook (31st ed.)
  - ISO 2768-1 (General tolerances for linear dimensions)
"""
from dataclasses import dataclass
from enum import Enum


class ISO2768Class(Enum):
    FINE = "f"
    MEDIUM = "m"
    COARSE = "c"
    VERY_COARSE = "v"


# ISO 2768-1 Table: tolerance in mm for linear dimensions
# Key: (class, (min_dim_exclusive, max_dim_inclusive))
ISO_2768_LINEAR: dict[ISO2768Class, dict[tuple[float, float], float]] = {
    ISO2768Class.FINE: {
        (0.5, 3): 0.05,   (3, 6): 0.05,    (6, 30): 0.10,
        (30, 120): 0.15,   (120, 400): 0.20, (400, 1000): 0.30,
        (1000, 2000): 0.50,
    },
    ISO2768Class.MEDIUM: {
        (0.5, 3): 0.10,   (3, 6): 0.10,    (6, 30): 0.20,
        (30, 120): 0.30,   (120, 400): 0.50, (400, 1000): 0.80,
        (1000, 2000): 1.20,
    },
    ISO2768Class.COARSE: {
        (0.5, 3): 0.20,   (3, 6): 0.30,    (6, 30): 0.50,
        (30, 120): 0.80,   (120, 400): 1.20, (400, 1000): 2.00,
        (1000, 2000): 3.00,
    },
    ISO2768Class.VERY_COARSE: {
        (0.5, 3): 0.50,   (3, 6): 0.50,    (6, 30): 1.00,
        (30, 120): 1.50,   (120, 400): 2.50, (400, 1000): 4.00,
        (1000, 2000): 6.00,
    },
}


# Process capability defaults (± tolerance for given process + dim range)
PROCESS_DEFAULTS: dict[str, dict[tuple[float, float], float]] = {
    "CNC_milling": {
        (0, 25): 0.025,  (25, 100): 0.038,  (100, 500): 0.075,
    },
    "CNC_turning": {
        (0, 25): 0.013,  (25, 100): 0.025,  (100, 500): 0.050,
    },
    "injection_molding": {
        (0, 25): 0.10,   (25, 100): 0.15,   (100, 300): 0.25,
    },
    "sheet_metal_bending": {
        (0, 50): 0.25,   (50, 200): 0.40,   (200, 1000): 0.80,
    },
    "die_casting": {
        (0, 25): 0.10,   (25, 100): 0.20,   (100, 500): 0.40,
    },
    "FDM_3D_print": {
        (0, 50): 0.20,   (50, 200): 0.40,   (200, 500): 0.80,
    },
    "SLS_3D_print": {
        (0, 50): 0.15,   (50, 200): 0.30,   (200, 500): 0.60,
    },
}


def get_iso2768_tolerance(
    nominal_dim: float,
    tolerance_class: ISO2768Class = ISO2768Class.MEDIUM,
) -> float:
    """Return ± tolerance (mm) per ISO 2768-1 for a given dimension."""
    table = ISO_2768_LINEAR[tolerance_class]
    for (lo, hi), tol in table.items():
        if lo < nominal_dim <= hi:
            return tol
    raise ValueError(
        f"Dimension {nominal_dim} mm out of ISO 2768-1 range "
        f"for class {tolerance_class.value}"
    )


def get_process_tolerance(
    nominal_dim: float,
    process: str = "CNC_milling",
) -> float:
    """Return ± tolerance (mm) for a manufacturing process."""
    if process not in PROCESS_DEFAULTS:
        raise ValueError(
            f"Unknown process '{process}'. "
            f"Available: {list(PROCESS_DEFAULTS.keys())}"
        )
    table = PROCESS_DEFAULTS[process]
    for (lo, hi), tol in table.items():
        if lo < nominal_dim <= hi:
            return tol
    raise ValueError(
        f"Dimension {nominal_dim} mm out of range for process '{process}'"
    )
```

```python name=reporting.py
"""
Generates visual and tabular stack-up reports.
"""
import matplotlib.pyplot as plt
import matplotlib.patches as mpatches
import numpy as np
from stack_up_graph import StackUpResult


def generate_stack_up_report(result: StackUpResult, save_path: str = None):
    """Create a multi-panel stack-up report figure."""
    fig, axes = plt.subplots(1, 3, figsize=(18, 6))
    fig.suptitle(f"Tolerance Stack-Up: {result.gap_name}", fontsize=14)

    # ── Panel 1: Contributor Waterfall Chart ──
    ax1 = axes[0]
    names = [c.name[:25] for c in result.contributors]
    nominals = [c.nominal * c.sensitivity for c in result.contributors]
    colors = [
        "#2196F3" if c.contributor_type.value == "internal"
        else "#FF9800" if c.contributor_type.value == "contact"
        else "#4CAF50"
        for c in result.contributors
    ]
    cumulative = np.cumsum(nominals)
    starts = np.concatenate([[0], cumulative[:-1]])

    ax1.barh(range(len(names)), nominals, left=starts,
             color=colors, edgecolor="white")
    ax1.set_yticks(range(len(names)))
    ax1.set_yticklabels(names, fontsize=8)
    ax1.set_xlabel("Cumulative Dimension (mm)")
    ax1.set_title("Dimension Chain (Waterfall)")
    ax1.axvline(x=result.nominal_gap, color="red", linestyle="--",
                label=f"Nominal Gap: {result.nominal_gap:.3f}")
    ax1.legend(fontsize=8)

    # ── Panel 2: Monte Carlo Histogram ──
    ax2 = axes[1]
    if result.monte_carlo_mean is not None:
        # Re-run MC for histogram data (or pass from result)
        ax2.set_title("Monte Carlo Distribution")
        ax2.set_xlabel("Gap (mm)")
        ax2.set_ylabel("Frequency")
        ax2.axvline(x=result.nominal_gap, color="red", linestyle="--",
                     label="Nominal")
        ax2.text(0.05, 0.95,
                 f"μ = {result.monte_carlo_mean:.4f}\n"
                 f"σ = {result.monte_carlo_std:.4f}\n"
                 f"Yield = {result.monte_carlo_yield:.2f}%",
                 transform=ax2.transAxes, fontsize=9,
                 verticalalignment="top",
                 bbox=dict(boxstyle="round", facecolor="wheat"))
        ax2.legend(fontsize=8)

    # ── Panel 3: Sensitivity Pareto ──
    ax3 = axes[2]
    if result.sensitivity_ranking:
        s_names = [s[0][:25] for s in result.sensitivity_ranking]
        s_pcts = [s[1] for s in result.sensitivity_ranking]
        cum_pct = np.cumsum(s_pcts)

        bars = ax3.barh(range(len(s_names)), s_pcts, color="#9C27B0")
        ax3.set_yticks(range(len(s_names)))
        ax3.set_yticklabels(s_names, fontsize=8)
        ax3.set_xlabel("% Variance Contribution")
        ax3.set_title("Sensitivity Pareto")

        ax3_twin = ax3.twiny()
        ax3_twin.plot(cum_pct, range(len(s_names)), "r-o", markersize=4)
        ax3_twin.set_xlabel("Cumulative %", color="red")

    plt.tight_layout()
    if save_path:
        plt.savefig(save_path, dpi=150, bbox_inches="tight")
    plt.show()


def print_summary_table(result: StackUpResult):
```

## 4. Where ML Adds Value (and Where It Doesn't)

This is critical: **the core stack-up math should NOT be ML-based.** Worst-case, RSS, and Monte Carlo are exact analytical/numerical methods. ML adds value in the *surrounding workflow*:

| Task | Best Approach | Why |
|---|---|---|
| **Stack-up arithmetic** | Analytical (WC/RSS) + Monte Carlo | Exact; no training data needed |
| **Identifying dimension chains from B-Rep** | Graph algorithms + rules | Deterministic, verifiable |
| **Auto-detecting mating faces** | **ML (GNN or point-cloud classifier)** | Heuristic proximity checks fail on complex assemblies; a trained model generalises |
| **Suggesting process-appropriate tolerances** | **ML (regression on historical data)** | Maps {feature type, material, process, volume} → achievable tolerance from shop floor data |
| **Predicting assembly yield from stack-ups** | **ML (gradient-boosted model)** | Captures non-linear interactions between multiple stack-ups at once |
| **Flagging high-risk stack-ups before analysis** | **ML (classification on past quality escapes)** | Learns from company-specific failure modes |
| **GD&T recommendation from design intent** | **NLP/LLM + rules** | Emerging; useful for drafting, not final specification |
| **Tolerance optimisation (cost vs. quality)** | Optimisation (genetic alg / Bayesian) | ML can provide the surrogate cost model; the optimiser is classical |

## 5. Data Requirements & Practical Constraints

| Requirement | Notes |
|---|---|
| **CAD format** | STEP AP242 preferred. AP214, Parasolid, IGES supported with reduced PMI fidelity |
| **Assembly structure** | Must include mate definitions or positioned components (not just a bag of parts) |
| **GD&T annotations** | Dramatically improves automation; without them, 60-80% of effort is manual tolerance input |
| **Training data for ML modules** | Mating face classifier: ~500+ labelled assemblies. Tolerance predictor: historical inspection data from MES/QMS |
| **Compute** | Monte Carlo (100k iterations, 20 contributors): <1 second on a laptop. CAD parsing: 2-10 seconds per assembly |
| **Deployment** | Desktop plugin (SolidWorks/Creo/NX API) or web app with STEP upload |

## 8. Concrete Next Step

**Start with Phase 1.** Build a working CLI tool that:

1. Reads a STEP file → extracts part geometry
2. Takes a manually-defined dimension chain (JSON/YAML config)
3. Computes WC, RSS, Monte Carlo
4. Outputs a sensitivity report

This validates the math engine and report format with real engineers before investing in the harder automation and ML layers. The `stack_up_graph.py` and `tolerance_defaults.py` modules above are functional starting points.

**First test case:** A simple 4-part linear stack (e.g., shaft + 2 spacers + housing) where the critical gap is end-play. This is textbook-verifiable and proves the pipeline end-to-end.
